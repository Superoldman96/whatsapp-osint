WhatsApp Beacon

Track WhatsApp presence sessions, store history in SQLite, and generate polished analytics dashboards.

Python 3.8+ | Selenium | MIT License
01

The Problem

  • No presence history. WhatsApp shows "online" in real time, but never records it.
  • Manual observation doesn't scale. You can't sit and watch a chat window 24/7.
  • Behavioral patterns are invisible. Without data, you can't see daily routines or anomalies.
  • No export, no analysis. WhatsApp offers zero analytics on when your contacts are active.
02

WhatsApp Beacon solves this

  • Automated polling. Selenium watches WhatsApp Web and detects online/offline transitions.
  • Session logging. Every online session gets a start time, end time, and duration in SQLite.
  • Runs headless. Authenticate once, then let it run in the background indefinitely.
  • Rich analytics. Generate a self-contained HTML dashboard with heatmaps, leaderboards, and filters.

How the Polling Loop Works

1
Launch Chrome via Selenium
Reuses saved profile from data/chrome_profile for session persistence.
2
Navigate to WhatsApp Web
Waits for login (QR scan on first run, auto-login after that).
3
Search for target contact
Types username into the search box, clicks the first matching result.
4
Poll the status header
Checks for localized "online" text. Transitions trigger session start/end events.
5
Write to SQLite
Session start, end, and computed duration stored in the Sessions table.
06

beacon.py: The Core Engine

  • WhatsAppBeacon class manages the entire Selenium lifecycle.
  • XPath candidates for login, search, and results adapt to WhatsApp DOM changes.
  • Prefers data-testid and aria-label over positional XPaths.
  • Supports both Lexical rich-text editor and native input variants.

analytics.py: Under the Hood

  • AnalyticsDashboard class reads the SQLite DB and generates a self-contained HTML file.
  • SQL JOIN merges Sessions + Users, filters out incomplete rows, orders by timestamp.
  • Data pre-processing in Python: daily aggregations, weekday/hour heatmap matrix, duration distribution buckets.
  • JSON embedding: processed data gets serialized directly into the HTML template as inline JavaScript objects.
  • Zero dependencies at runtime. The output HTML uses no CDN, no external libs.

XPath Resilience

  • WhatsApp Web updates its DOM frequently. Positional XPaths break on every redesign.
  • The beacon uses candidate lists of 5+ XPaths per element (login, search, results).
  • Prefers data-testid and aria-label attributes: they survive layout changes better than positional paths.
  • Supports both the Lexical rich-text editor and native input variants that WhatsApp alternates between.
  • When selectors break, fix the 3 candidate arrays in beacon.py: _LOGIN_READY_XPATHS, _SEARCH_BOX_XPATHS, _SEARCH_RESULT_XPATHS.
